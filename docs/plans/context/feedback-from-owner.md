1. 日常流程
   • 完整服务流程：客人首先在平台上完成下单。店方通常会提前通过电话号码搜索并添加客人的 WhatsApp（或微信、Line、KakaoTalk）建立联系。建群后，店方会与客人确认订单详细信息、航班号以及具体人数。等客人抵达当天，店方会前往机场接机并将客人送至门店取车。

• 最花时间的工作：店方明确表示，联系不上客人是最耗费时间的环节；此外，去机场接送客人也是业务中最麻烦、最花时间的实际操作步骤。

2. 预约与排期
   • 记录方式：目前主要依赖人工手动在 Excel 表格上记录这40到50台车的排期情况。
   • 排期冲突：由于是多平台挂靠且依赖人工记录，录音中提到需要“视觉化”的系统来锁定库存，以防某台车被占用的情况下再次被派单（爆单或重复预约）。

• 取消与爽约处理：店方有明确的阶梯式扣费规则。如果距离取车72小时以上取消，不收费用。在72到48小时内取消扣除30%；48到24小时内扣除70%；如果是当天取消，则扣除100%的全款。

3. 客人验证
   • 检查证件：必须检查客人的护照、本国驾照以及国际驾照。
   • 证件问题：经常遇到问题，例如国际驾照已经过期。此外，店方严格要求必须持有符合“1949年日内瓦公约”的国际驾照，像中国大陆、缅甸、菲律宾等非该公约缔约国或使用其他版本公约的驾照，店方会直接拒租。

4. 车辆管理
   • 追踪空车：目前是通过 Excel 表格来实现“视觉化”，即在表格上标出日期（如1号到5号），把某台车对应的日期单元格“占死”，以此来判断哪些车有空。
   • 车况检查：店方认为传统的纸上记录不够详细，他们倾向于通过系统上传车辆的照片或视频来记录车况。这样操作完成后，客人和店方都能直观地看到车辆的原始状态。

5. 收款与纠纷
   • 收费与便利性：客人一般需要在店内刷卡支付，店方认为客人使用现金非常不方便。如果是发生事故，店方会先向客人收取15万日元的费用（NOC及维修相关垫付）。
   • 纠纷与处理：确实会发生纠纷，例如客人发生刮蹭、撞车事故，或者出现不付钱、把车扔了跑路的情况。为了应对这些问题（特别是防范偷车或弃车），店方在车上安装了 GPS 定位系统，以便随时追踪车辆位置并协助报警处理。

6. 沟通
   • 主要语言：外国客人主要使用英语沟通，其次是中文。虽然也有泰国或韩国客人，但由于店方不懂泰语或韩语，大家通常还是默认使用英语交流。
• 沟通导致的常见问题：最大的沟通障碍发生在车辆出事故时。由于海外客人不懂日语且不熟悉当地地理，在需要报警或呼叫救援时，他们往往无法向警察或店方准确描述自己所在的地点。此时店方只能依靠车上的 GPS 来确定客人的准确位置并提供帮助。

💡 最关键的一个问题
如果只能改一件事让工作轻松很多，最想改什么？
• 答案是：引入自动化的车辆排期与库存管理系统（实现“视觉化”）。

• 原因：店主目前对 Excel 手动排期的现状感到吃力，他最迫切的需求是拥有一个像 Google Calendar 或者专用租车系统一样的工具。这个系统需要能直观显示哪台车在哪天被租用、哪天有空，并在订单生成时自动锁定库存，从而彻底解放他们每天手动翻查表格、核对排期的时间。
